# Purpose

The OpenCharge project is committed to fostering an open, professional, and inclusive community where everyone can collaborate, learn, and contribute.

This Code of Conduct establishes the expected standards of behaviour for all contributors, maintainers, users, and community members.

---

# Our Commitment

We are committed to creating an environment that is:

* Respectful
* Professional
* Inclusive
* Collaborative
* Constructive
* Safe for learning and innovation

Everyone participating in the OpenCharge community should feel welcome regardless of experience level, background, or geographic location.

---

# Expected Behaviour

Community members are expected to:

* Treat everyone with respect and professionalism.
* Be welcoming to new contributors.
* Share knowledge openly.
* Accept constructive feedback.
* Focus discussions on technical and project-related topics.
* Communicate clearly and respectfully.
* Help improve documentation and code quality.
* Follow project guidelines and engineering standards.

---

# Unacceptable Behaviour

The following behaviours are not acceptable:

* Harassment or discrimination of any kind.
* Personal attacks or insults.
* Trolling or intentionally disruptive behaviour.
* Offensive language or inappropriate content.
* Publishing private or confidential information without permission.
* Threats, intimidation, or abusive behaviour.
* Spam or repeated off-topic discussions.
* Deliberate attempts to damage the project or community.

---

# Engineering Culture

OpenCharge is an engineering-focused project.

We value:

* Technical excellence
* Continuous learning
* Well-documented solutions
* Clean architecture
* High-quality code
* Safety-first design
* Constructive technical discussions

Disagreements are natural in engineering. They should always be resolved respectfully and supported by technical reasoning.

---

# Collaboration Principles

We encourage contributors to:

* Ask questions.
* Review code respectfully.
* Suggest improvements.
* Share ideas openly.
* Document design decisions.
* Learn from one another.

Every contribution—whether code, documentation, testing, hardware design, or bug reporting—is valuable.

---

# Reporting Issues

If you experience or witness behaviour that violates this Code of Conduct, please report it privately to the project maintainer.

When reporting an issue, include:

* A description of the incident.
* Relevant screenshots or logs (if applicable).
* The date and location of the incident.
* Any additional context that may help.

Reports will be handled respectfully and confidentially whenever possible.

---

# Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Depending on the severity of a violation, actions may include:

* Friendly reminder.
* Request to modify behaviour.
* Removal of inappropriate content.
* Temporary suspension from project participation.
* Permanent ban from the community.

All enforcement decisions will be made with fairness and professionalism.

---

# Scope

This Code of Conduct applies to all OpenCharge community spaces, including:

* GitHub repository
* GitHub Discussions
* GitHub Issues
* Pull Requests
* Documentation
* Community meetings
* Project-related communication channels

---

# Our Philosophy

OpenCharge is more than a software repository.

It is a long-term engineering initiative dedicated to advancing open-source electric vehicle charging technology through collaboration, transparency, and professional engineering practices.

We believe that the best engineering solutions are created through respectful collaboration and continuous learning.
