# Versioning Policy

OpenCharge follows Semantic Versioning.

* **Major** – Breaking architectural or API changes.
* **Minor** – New features and functionality.
* **Patch** – Bug fixes, documentation updates, and minor improvements.

Example:

```text
1.2.3

1 = Major
2 = Minor
3 = Patch
```
